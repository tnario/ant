API Reference

useRoutes(): [ Route[], RouteController ]

returns a tuple that contains two variables:

container: Route[] - holds a list of the routes that have been pushed by the controller.

controller: RouteController - and object that has methods for adding routes to the container by referance.

router(prefix: string, ...middleware: CallBack[])(routes: Route[] = []): RouterContainer

this returns a router container, that holds the prefix, an array of Router-Level middleware and an array of routes.

The first invokation brackets requires:

prefix: string - a path prefix for identification.

...middleware: CallBack[] - an array of Router-Level functions.

The second invokation brackets requires:

routes: Route[] - an array of routes. Default is an empty array.

useMiddleware(...middleware: CallBack[]): CallBack[]

returns the same middleware array. This funtion is only for readability. CallBack funtions can be inserted in Application-Level, Router-Level and Route-Level handling.

useErrorMiddleware(...middleware: ErrorCallBack[]): ErrorCallBack[]

returns the same middleware array. This funtion is only for readability. ErrorCallBack is used only for error handling purpose.

async serveHTTP(...)

starts the web server process.

serveHTTP is a three stage higher-order function, the first invocation accepts three arguments:

middleware: CallBack[] - is used to set Application-Level functions.

routes: Route[] - used to set any additional routes to the application.

error: ErrorCallBack[] - is used for the error handling process and is required.

The second invocation is for adding routers (RouterContainer[])

The last invocation accepts the address and an optional calback for the server:

addr: string | HTTPOptions

cb?: () => void

async serveHTTPS(...)

almost identical to serveHTTP except the argument for addr is diffrent and requires additional information for running a secure web server (TLS).
